## 学习与实践记录（docs/learn）

本目录用于沉淀在本项目开发与运维实践中的工程经验、踩坑记录、以及复盘与改进方案。目标是：

- 快速复用成熟方案，少走弯路
- 为团队成员提供清晰的知识入口
- 将“现象—诊断—方案—验证—教训”过程标准化

### 与全局流程的联动
1. **任务前复盘**：在编写 `IMPLEMENTATION_PLAN.md` 之前，搜索是否有相似主题的 learn 文档，将相关经验纳入风险与假设部分。
2. **调试中记录**：发现新的根因或关键排查手法时，及时起草草稿文档，并在修复完成后补全结构化内容。
3. **交付后沉淀**：代码合入后，回填最终验证结果与后续改进计划；在任务总结中引用该文档路径。
4. **任务闭环**：如 learn 文档影响现有 `docs/tasks/*.md` 简报或 `knowledge_base` 参考材料，追加引用或更新链接，保持知识一致性。

### 如何撰写
- 新的经验或复盘：在本目录下新增一个 Markdown 文件，命名建议包含日期与主题。例如：`2025-09-11-render-vite-ws-nginx.md`
- 在文档中结构化记录：背景、问题、诊断、改动、验证、结果、教训、后续改进
- 为关键修改添加代码或配置片段引用，确保可复制粘贴复现
- 使用“References”小节列出关联的任务简报、实现计划、提交记录等

### 约定
- 文档面向工程师，强调可操作性与关键决策点
- 示例优先真实产线场景，避免过度理论化
- 如果经验已沉淀为长期规则或模式，请在相应 `knowledge_base` 或 `developer_guides` 中补充引用

_让 learn 目录成为问题排查与经验传承的第一站。_
